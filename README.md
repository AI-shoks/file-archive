# file-archive

Личный органайзер учебных материалов. Загрузка файлов с ручной меткой
предмета и хранением по папкам.

Границы и архитектурные решения — в [CLAUDE.md](CLAUDE.md).
Порядок работ — в [ROADMAP.md](ROADMAP.md).

Тесты прогоняются в CI (GitHub Actions) на каждый push и pull request — см.
[.github/workflows/ci.yml](.github/workflows/ci.yml).

## Запуск локально

```bash
python -m venv venv
venv\Scripts\activate            # Windows; на *nix: source venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload
```

Тесты:

```bash
python -m pytest
```

## Веб-интерфейс (Слой 2)

После запуска открой корень `/` (локально http://127.0.0.1:8000/) — там
статический фронтенд поверх API: список предметов, файлы выбранного предмета
со скачиванием, загрузка файла и поиск по имени. Это vanilla HTML/JS/CSS без
сборки ([static/](static/)), который FastAPI отдаёт сам — отдельного
фронтенд-деплоя нет.

## Поведение при загрузке

- Если файл с таким именем уже есть в папке предмета, новый сохраняется
  с суффиксом (`report.docx` → `report(1).docx` → `report(2).docx`), а не
  затирает существующий. Ответ возвращает фактическое (итоговое) имя файла.

## Известные ограничения (демо, Слой 1)

- **Запись публичная и без авторизации** by design (Слой 1). На публичном
  URL это значит, что заливать файлы в засидированные предметы может кто
  угодно — барьер (rate-limit/токен) вне скоупа Слоя 1.
- **Лимит размера проверяется после буферизации файла в RAM** — защищает
  диск, но не от OOM при больших/параллельных заливках (см.
  [storage.py](storage.py), комментарий к `MAX_FILE_SIZE_BYTES`).

## Переменные окружения

| Переменная           | Назначение                                              | Дефолт                            |
|----------------------|---------------------------------------------------------|-----------------------------------|
| `FILE_ARCHIVE_ROOT`  | Корневой каталог хранилища                               | `C:/Users/artem/file-archive-data`|
| `SEED_SUBJECTS`      | CSV-список папок-предметов, создаваемых при старте       | пусто (сидирование не выполняется)|

## Деплой (Render)

Конфигурация — [render.yaml](render.yaml). Старт:
`uvicorn main:app --host 0.0.0.0 --port $PORT`.

**Холодный старт.** На free-tier Render инстанс засыпает; первый запрос
после простоя может занять ~50 секунд, пока сервис просыпается. Это не
поломка — подождите ответа.

**Хранилище на демо-деплое эфемерное by design.** Диск Render очищается
при каждом редеплое и засыпании инстанса, поэтому загруженные файлы не
сохраняются между деплоями, а папки-предметы пересоздаются при старте из
`SEED_SUBJECTS`. Для постоянного хранения достаточно подключить Render
Persistent Disk и указать `FILE_ARCHIVE_ROOT` на его mount path — замена
тривиальна и не требует изменений кода.
