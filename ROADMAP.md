# ROADMAP

Последовательность работ. Про *границы и архитектуру* — см. CLAUDE.md;
здесь только *порядок*. Принцип очередности: сначала то, без чего нет
демо (деплой), потом минимальный полезный цикл чтения, потом тесты,
и только в конце — опциональные фичи.

- [x] Этап 1: загрузка файла на диск (`POST /upload/file`)

- [x] Этап 1.2: тесты эндпоинта `/upload/file` (TestClient)

## План

1. **Деплойный блокер (приоритет №1).**
   Вынести `ROOT_DIR` в переменную окружения, задеплоить на Render.
   Формально вне скоупа Слоя 1, но идёт первым: без задеплоенного
   приложения нет демо. Цель — рабочий публичный URL.

2. **Минимальный цикл чтения.**
   - `GET /subjects` — список существующих предметов (папок в `ROOT_DIR`)
   - `GET /files/{subject}` — список файлов в предмете
   - скачивание конкретного файла (`GET /files/{subject}/{filename}`)
   Это минимум, после которого «органайзер» уже что-то показывает,
   а не только принимает загрузки.

3. **Тесты цикла чтения.**
   Покрыть эндпоинты из пункта 2 (TestClient + изоляция ФС, как в
   `tests/test_main.py`).

4. **Поиск (опционально, НЕ блокирует демо).**
   `GET /search?q=...` — поиск по имени файла. Заявлен в скоупе Слоя 1,
   но идёт хвостом: демо работает и без него.

## Цель

Через ~2 недели — задеплоенный, протестированный, демонстрируемый
проект с публичным URL, а не полный Слой 1 на localhost.
